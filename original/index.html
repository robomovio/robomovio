Defining attributes

For the robot

We first think about these three main parts:

  • How should the robot move?
  • How should the robot interact with the environment?
  • How should the robot detect obstacles / sense other influences?

1. Movement

Well, for moving, the robot needs wheels. The wheels have to be powered by motors. So, we could either use a

  • tank driven robot (2 wheels, next to each other and a wheel / marble, stabilizes the robot)
  • or a robot with a steering like a car (2 wheels at the back (move the robot forwards) and 1 - 2 wheels steering)

We chose the first approach because it only needs two wheels and a marble (which are included in the standard spike prime set) and because the other option is very difficult to build.

2. Interaction

For the interaction, the robot needs motors + specific arms or mechanical mechanisms to solve the stations. Every station is different and we can only use up to 4 motors. So it's very important to reuse the motors.

We already use 2 motors for moving so we can effectively use the 2 other motors. There are 2 ways:

  • Build one arm which could solve all tasks
  • Build different extensions which use a specific interface to connnect to the robot and the motors

The second approach is definitely the best because these extensions can vary between the task and exactly fit with it.

3. Sense

Let's assume the extension motors have a lot of friction and the task isn't aligned properly to the map. The ground could be very unconsistent and slippery to.
If you now run your working code on the robot, it will completely mess up. So what to do?

Use sensors!

Following sensor types are available for us:

  • Touchsensor:
    (detects pressure on the button at the front)
  • Colorsensor
    (detects colors, light reflection / brightness)
  • Distance- / Ultrasonicsensor:
    (measures distance / Sends and recieves signals with ultrasonic frequencies)
  • Gyrosensor:
    (measures angular velocity, motion and orientation)

We can use any combination and count of these sensors.
But:
We already use 4 motors and the spike prime hub has just 6 ports, so 2 connections left for sensors.
We don't need an external Gyro because the Spike Prime has one integrated. After a bit of brainstorming it became clear that we should use two color sensors for aligning the robot at lines next to tasks and to follow lines. But we're still not sure if we should attach each next to one wheel for alignment or at the front of the robot to follow lines.
